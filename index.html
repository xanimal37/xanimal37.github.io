Why are there so many different ways to say "equals" or "is equal to" in Java? The reason is because there are different ways objects can be equivalent to one another.

Java types

There is more than one entity type in Java. Primitive types include byte, short, int, long, float, double, boolean and char. Non-primitive types include String, Array, Object, and just about anything else.

objects

Object is the class that all classes inherit from. This includes the String class. This is why String is an object and not a primitive, as it might seem to be. Because they derive from the Object class, all objects include a method called "equals".
