what is equal?

Why are there so many different ways to say "equals" or "is equal to" in Java? Although these all seem very similar, they serve very different purposes in Java. The "=" character is actually an assignment operator, that is it peforms an operation which set something as equal to something else, as in the following variable assignment:

int numDogs = 2;

.equals() and "==" are ways of comparing two entities, but they differ in use.

Java types

There is more than one entity type in Java. Primitive types include byte, short, int, long, float, double, boolean and char. Non-primitive types include String, Array, Object, and just about anything else.

objects

In Java, all classes inherit from Object. This includes the String class. This is why String is an object and not a primitive, as it might seem to be. Because they derive from the Object class, all objects created from a class include a method called "equals". In fact, all classes also inherit a method called "toString" that will output a string representation of the object. By default this is the address of the object in memory. This will become useful in examining the difference between .equals() and == in the case of non-primitives like objects.

If we test two primitives with "==", the result is straightforward:

System.out.println(4==4);
...
true

If we create two instances of a class, as in the following example, let's see what happens when we test for equality using "==".

public class Animal {
public String kingdom = "Animal";
}

Animal dog = new Animal();
Animal cat = new Animal();

System.out.println(dog == cat);

The output will be "false". Even though these two Animal objects have the same properties and so could be considered equivalent to each other, if we use the toString() method that all objects have we can see that the addresses are different. And that is why these two objects are not considered equal - they are different objects in memory.

Animal@1cd072a9
Animal@7c75222b

and if we try

System.out.println(dog.equals(cat));

we get

false

So what is the point if we get the same result using the method equals() or ==?

One thing we can do with classes is override methods. This means that a class that inherits a method can have its own version of the method. So we can determine what makes two objects equivalent when we compare them. If we override the equals() method in the Animal class:

boolean equals(Animal other){
if(other.kingdom.equals(kingdom)){
return true;
}
else {
return false;
}
}

Now the output will be "true" because we are comparing the two objects based on one property, which they happen to have in common. This is much more useful than the default method.
